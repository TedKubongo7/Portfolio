About Me

The person behind the work.

I'm a supply chain data analyst based in Calgary with experience building reporting tools, automation workflows, and business applications. My work combines hands-on operational context with practical software development.

My advantage is that I have worked inside the kinds of environments I now build for. I've seen firsthand what broken spreadsheets, manual document handling, inconsistent data, and slow reporting actually cost a team — in time, accuracy, and decision quality. I build tools around those real problems, not abstract use cases.

At one organization, I automated an end-to-end supply chain workflow that had previously taken multiple days to run manually. The rebuilt system completed the same process in minutes, with better accuracy and full auditability. That experience made one thing clear: most organizations are sitting on solvable problems — they just haven't had someone build the solution yet.

Through Kai Technologies, I build automation and applied AI solutions for mid-market companies across North America. My core stack includes Python, FastAPI, and the Anthropic Claude API, with a focus on tools that support decisions, organize data, and reduce manual effort.

I hold a Bachelor of Arts from UBC and completed Computer Science coursework at the University of Saskatchewan. I also hold a Microsoft AI & Machine Learning certificate.

Case Study 01 · Supply Chain Operations
End-to-End Data Pipeline Automation
Replacing a multi-day manual workflow with a fully automated system
Days → Minutes15h+/week savedZero manual uploads
The Problem

Analysts were manually generating files from multiple data sources, running repetitive validation checks in Excel, uploading files into internal systems by hand, and troubleshooting errors with no visibility into what failed or why. The process took multiple days per cycle, was error-prone, and impossible to scale.

The Solution
  • Automated file generation from source data using Python
  • Multi-level validation with duplicate detection and business rule checks
  • Browser automation for system uploads, including SSO-authenticated workflows
  • Excel automation for pivot summaries, issue tracking, and structured output
  • End-to-end pipeline execution requiring minimal human intervention
The Impact
  • Processing time reduced from multiple days to minutes
  • 15+ hours of manual work eliminated per week
  • Data accuracy and consistency improved significantly
  • Operations became fully repeatable and scalable
  • Errors surfaced automatically, with clear visibility into root cause

Case Study 02 · Data Quality & Reporting
Automated Data Quality Monitoring & Executive Reporting
Weekly trend visibility for leadership — without the manual effort
Manual reporting eliminatedLeadership visibilityTrend tracking enabled
The Problem

Data quality issues across supply chain datasets were tracked manually with no centralized view. There was no way to identify recurring problems over time, no consistent classification, and reporting required repetitive Excel manipulation every week. Leadership lacked any clear picture of issue trends or root causes.

The Solution
  • Automated weekly data aggregation across quality outputs
  • Standardized issue classification logic (Issue vs. No Issue)
  • Centralized tracker updated automatically each cycle
  • Pivot-based reporting showing trends, root causes, and historical patterns
  • Consistent output format ready for leadership review without manual prep
The Impact
  • Weekly manual reporting effort fully eliminated
  • Recurring issues identified and tracked over time
  • Leadership gained clear, consistent visibility into data health
  • Reporting standardized across the team with no additional effort
  • Enabled data-driven decisions based on trends, not one-off snapshots

Case Study 03 · Automation Framework
Reusable Python + Excel Automation Framework
Building once to solve many — a modular foundation for repeatable workflows
Reusable across workflowsReduced Excel dependencyFaster future builds
The Problem

Teams relied heavily on manual Excel work for data transformation, validation, and reporting. Each workflow was built from scratch, creating inconsistency and making future automation slow to deliver. There were no shared building blocks — every new request started at zero.

The Solution
  • Designed a modular hybrid system combining Python (pandas) with Excel (xlwings)
  • Reusable functions for column transformations, concatenation, and validation
  • Multi-stage validation processes applicable across different datasets
  • Dynamic pivot table creation built into the framework
  • Documented and structured for team adoption and future extension
The Impact
  • Manual Excel dependency significantly reduced across the team
  • New automation workflows delivered faster using shared components
  • Consistent output quality regardless of who ran the process
  • Framework extended to support multiple use cases without rebuilding from scratch
